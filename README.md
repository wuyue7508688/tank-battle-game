# 局域网坦克大战

一个可以在公司局域网内联机玩的浏览器坦克大战小游戏。

玩家打开同一个局域网地址，输入昵称后进入大厅，可以创建房间、加入房间、选择红蓝队，并进行团队对战。

## 技术栈

- 前端：HTML、CSS、JavaScript、Phaser 3
- 后端：Node.js、Express、Socket.IO
- 数据保存：服务器内存

服务器重启后，房间、玩家状态和本局战绩都会清空。

## 启动方式

第一次运行：

```bash
npm install
npm start
```

以后已经安装过依赖时：

```bash
npm start
```

启动成功后，终端会显示类似：

```text
本机访问地址: http://localhost:3000
局域网访问地址: http://192.168.x.x:3000
```

自己测试时打开本机访问地址。

同一个局域网内的其他玩家打开局域网访问地址。

停止服务器：

```text
Ctrl + C
```

## 游戏流程

1. 打开网页。
2. 输入昵称。
3. 进入大厅。
4. 创建房间或通过房间码加入房间。
5. 选择红队或蓝队。
6. 房主点击开始游戏。
7. 3 秒倒计时后开始战斗。
8. 达到胜利条件后进入结算页。
9. 房主可以点击再来一局。

## 操作方式

- `W` `A` `S` `D` 或方向键：移动坦克
- 鼠标：控制炮口方向
- 鼠标左键：开火
- 按住 `Tab`：显示记分板

## 房间规则

- 每个房间最多 8 人。
- 每队最多 4 人。
- 至少 2 人才能开始。
- 红队和蓝队都至少需要 1 人。
- 房主可以开始游戏。
- 游戏中禁止新玩家加入。
- 游戏结束后，新玩家可以加入房间，但只能参与下一局。

## 胜利模式

支持两种模式：

- 分数制：率先达到目标分数的队伍获胜。
- 时间制：时间结束时，分数更高的队伍获胜；同分显示平局。

## 常见问题

### 其他电脑打不开局域网地址

可能原因：

- 不在同一个局域网。
- 防火墙拦截了端口 `3000`。
- 公司网络限制了设备之间互访。

可以尝试关闭本机防火墙限制，或换一个端口启动：

```bash
PORT=3001 npm start
```

### 页面能打开，但联机不同步

先确认所有玩家打开的是同一个局域网地址。

如果服务器重启过，原来的房间会被清空，需要重新创建房间。

### 测试失败

先重新安装依赖：

```bash
npm install
```

再运行：

```bash
npm test
```
